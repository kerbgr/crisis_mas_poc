Supported File Formats
Expert Profiles

Nested Format:

{
  "agents": [
    {
      "agent_id": "medical_expert_01",
      "name": "Dr. Name",
      "role": "Medical Expert",
      ...
    }
  ]
}

Flat Format:

[
  {
    "id": "expert_001",
    "name": "Name",
    "role": "Role",
    ...
  }
]
Incident Protocols

Q&A Format:

[
  {
    "id": "protocol_001",
    "question": "How to...",
    "answer": "Step 1...",
    "category": "firefighting",
    "expert_id": "expert_001",
    ...
  }
]
Crisis Scenarios

Scenario Format:

{
  "id": "fire_001",
  "type": "wildfire",
  "name": "Fire Emergency",
  "description": "...",
  "available_actions": [...],
  "expert_selection": {...}
}

Each scenario is a separate .json file in the directory.

All formats are automatically detected and supported by the web tools. Paths can be absolute or relative to the project root.
